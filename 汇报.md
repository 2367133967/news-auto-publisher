## 一、项目目标

> 搭建端到端自动化工作流：用户发送新闻纯文本 → Agent 自主提取信息 → 精准修改数据文件 → Git 版本推送 → 网站静默更新。

---

## 二、系统架构

**数据流向**：用户输入 → Agent → exec工具 → Python脚本 → JSON文件 → 网站展示

**核心设计理念**：数据与表现分离 + 语义与执行解耦

| 层级 | 组件 | 职责 |
| :--- | :--- | :--- |
| 语义层 | Agent + SOUL.md | 理解意图，提取信息，调度工具 |
| 执行层 | 自动发布.py | 精准修改 JSON 数据文件 |
| 同步层 | git_sync.bat + 计划任务 | 异步版本控制，与交互解耦 |
| 表现层 | index.html | 动态渲染，实现静默重构 |

---

## 三、核心实现

### 3.1 Agent 配置（SOUL.md）
- 极简指令，强制调用 exec 工具，杜绝幻觉
- 命令：`python D:\project1\news-publisher\自动发布.py --title "{标题}" --content "{正文}" --author "{作者}"`

### 3.2 Python 脚本（自动发布.py）
- 读取 `news_list.json` → 插入新条目 → 写回文件
- 参数化输入（argparse），防命令注入
- 仅修改 JSON 文件，HTML/CSS/JS 零触碰

### 3.3 前端页面（index.html）
- 通过 `fetch()` 动态读取 `news_list.json` 并渲染
- 刷新即更新，无需重新部署

## 四、关键难点与解决

| 难点 | 解决方案 |
| :--- | :--- |
| exec 工具不执行 | 系统化排查，定位 `exec-approvals.json` 格式问题 |
| Git 推送致 Agent 超时 | 解耦架构：Agent 只发布，Git 异步同步 |
| SSH 22 端口被封锁 | 配置 SSH 走 443 端口 |
| 网页无法加载 JSON | 启动本地 HTTP 服务器绕过 CORS |
