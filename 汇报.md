# 基于大模型Agent的网站自动化发布与更新系统——项目汇报

**汇报人**：杨力行 
**日期**：2026年4月21日

---

## 一、项目目标

> 搭建端到端自动化工作流：用户发送新闻纯文本 → Agent 自主提取信息 → 精准修改数据文件 → Git 版本推送 → 网站静默更新。

---

## 二、系统架构

用户输入 → OpenClaw Agent → exec工具 → Python脚本 → news_list.json → 网站展示
      ↓
git_sync.bat
      ↓
GitHub仓库


**核心设计**：数据与表现分离 + 语义与执行解耦

| 层级 | 组件 | 职责 |
| :--- | :--- | :--- |
| 语义层 | Agent + SOUL.md | 理解意图，提取信息，调度工具 |
| 执行层 | 自动发布.py | 精准修改 JSON 数据文件 |
| 同步层 | git_sync.bat + 计划任务 | 异步版本控制，与交互解耦 |
| 表现层 | index.html | 动态渲染，实现静默重构 |

---

## 三、核心实现

### 3.1 Agent 配置（SOUL.md）

调用 exec 工具执行 python Dproject1news-publisher自动发布.py --title {标题} --content {正文} --author {作者}
极简指令，强制调用工具，杜绝幻觉

### 3.2 精准修改与安全约束
只修改 news_list.json，HTMLCSSJS 零触碰

参数化输入（argparse），防 Prompt 注入

审批机制（approve），防越权操作

### 3.3 静默重构与自动化更新
前端动态读取 JSON，刷新即更新，无需重新部署

Git 同步由计划任务每小时自动执行，与 Agent 完全解耦

## 四、关键难点与解决

### 4.1 难点清单

| 难点 | 解决方案 |
| :--- | :--- |
| **exec 工具不执行** | 系统化排查，定位 `exec-approvals.json` 格式问题 |
| **Git 推送致 Agent 超时** | 解耦架构：Agent 只发布，Git 异步同步 |
| **SSH 22 端口被封锁** | 配置 SSH 走 443 端口 |
| **网页无法加载 JSON** | 启动本地 HTTP 服务器绕过 CORS |

### 4.2 典型问题排查流程

1. 确认 Agent 是否真正调用工具（观察 `Tool call exec` 日志）
2. 检查 `openclaw config get tools.exec` 权限配置
3. 检查 `~/.openclaw/exec-approvals.json` 审批文件
4. 手动执行脚本验证功能正常
5. 查看 `openclaw logs --follow` 实时日志
